# 如何贡献

欢迎为 AISIP 协议贡献内容。

## 贡献方式

- **规范** — 通过 Issue 或 PR 提出协议变更建议
- **参考实现** — 添加其他语言的实现（Go、JS、Java 等）
- **文档** — 改进文档、添加教程、修复错别字
- **测试用例** — 为边界情况添加测试场景

## 贡献流程

1. **Fork** 本仓库
2. 从 `main` **创建功能分支**
3. **完成修改**，确保所有测试通过
4. **提交 Pull Request**，清晰描述改动内容

## 贡献原则

- 保持协议简洁 — 复杂性是缺陷，而非特性
- 参考实现不得引入外部依赖
- 所有控制流模式必须有对应测试用例
- 协议变更需同步更新规范文件和参考实现

## 运行测试

```bash
cd reference/python
python test_all.py
```

提交前所有测试必须通过。

## 行为准则

请保持尊重、建设性，专注于让协议对所有人更好用。

## 公理约束

所有贡献必须符合**公理 0：人类主权与福祉**。任何损害人类健康、尊严或自由的变更将不予受理。
